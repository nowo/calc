# API Overview

All exports from `@wzo/calc`, grouped by purpose:

## Expression / Formatting

- [`calc(expr, options?)`](/api/calc) — for computation: expression evaluation + (optional) formatting, **throws on error**
- [`fmt(value, options?)`](/api/fmt) — for display: a display-safe version of calc (supports arithmetic + formatting), **falls back to `_error` on error**

## Chaining

- [`chainAdd / chainSub / chainMul / chainDiv`](/api/chain) — chainable arithmetic, e.g. `chainAdd(10).sub(3).mul(2)()`

## Standalone Functions

- [`add / sub / mul / div`](/api/standalone) — return `number`
- [`addStr / subStr / mulStr / divStr`](/api/standalone) — return high-precision `string`

## Aggregation

- [`calcSum / calcAvg / calcMax / calcMin`](/api/aggregate) — array / object-array aggregation

## Global Config

- [`setConfig / resetConfig / getConfig`](/api/config) — global: error fallback, default format, division precision
- Per-call precision: `div` / `divStr` / `chainDiv` / `calcAvg` accept [`{ _precision }`](/api/config) as a trailing argument to override precision for that call only
