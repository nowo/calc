# `calc(expr, options?)`

The main entry function. Parses an expression string, evaluates it, optionally formats the result, and returns a string or number.

::callout{type="warning"}
**`calc` is for computation: errors (invalid expressions, unknown functions, etc.) throw immediately** — handle them with try/catch. For display scenarios where you need an error fallback, use [`fmt`](/api/fmt) instead — it has the same API, supports arithmetic just the same, but falls back to `_error` on error. **Use `calc` for computation, `fmt` for display.**
::

## Signature

```ts
calc(expr: string, options?: ICalcOptions): string | number

interface ICalcOptions {
    // true ⇒ % in expressions is treated as a unit marker (0.1% stays 0.1, output includes %)
    // false (default) ⇒ % means divide by 100
    _unit?: boolean

    // explicit format: an IFormat options object
    _fmt?: IFormat

    // division precision (defaults to global _precision = 50)
    _precision?: number

    // step-by-step debug: true ⇒ prints to console; pass a function ⇒ receives IDebugInfo via callback
    _debug?: boolean | ((info: IDebugInfo) => void)
}
```

> Errors throw immediately — there is no `_error` option. Use [`fmt`](/api/fmt) if you need a fallback.

## Usage

### Basic Arithmetic

```ts twoslash
import { calc } from '@wzo/calc'
// ---cut---
calc('0.1 + 0.2') // "0.3"
calc('(1 + 2) * 3') // "9"
calc('-5 + 3') // "-2"
```

### Use Template Interpolation Instead of Variables

`calc` **does not support variables** — embed values directly in the expression via template interpolation. Values remain visible and easy to debug:

```ts twoslash
import { calc } from '@wzo/calc'
// ---cut---
const a = 1
const b = 2
calc(`${a} + ${b}`) // "3"

const price = 99.99
calc(`${price} * 1.07`) // "106.9893"
```

::callout{type="info"}
Expressions are **pure arithmetic**: only the four operations, parentheses, and math functions. **Write conditional logic in JS outside the expression** rather than embedding it:

```ts twoslash
import { calc } from '@wzo/calc'
declare const a: number
// ---cut---
// 10% discount when order exceeds 100
const final = a > 100 ? calc(`${a} * 0.9`) : String(a)
```

For conditional value selection you can also use `clamp` / `min` / `max` inside expressions.
::

### Built-in Math Functions

A set of math functions can be used inside expressions, computed with BigInt precision primitives:

| Function | Description |
| :--- | :--- |
| `abs(x)` | Absolute value |
| `sign(x)` | Sign (-1 / 0 / 1) |
| `floor(x)` `ceil(x)` | Floor / ceiling (same as `Math.floor`/`Math.ceil`, including negatives) |
| `round(x)` `trunc(x)` | Round / truncate toward zero (same as `Math.round`/`Math.trunc`) |
| `sqrt(x)` | Square root (`x >= 0`; rounded to the division precision) |
| `pow(x, n)` | Exponentiation (`n` must be an integer; negative exponents use division) |
| `mod(a, b)` | Modulo (remainder sign matches the dividend) |
| `min(...)` `max(...)` | Minimum / maximum (variadic) |
| `clamp(x, lo, hi)` | Clamp to `[lo, hi]` |

```ts twoslash
import { calc } from '@wzo/calc'
// ---cut---
calc('pow(2, 10)') // "1024"
calc('mod(10, 3)') // "1"
calc('sqrt(2)', { _precision: 6 }) // "1.414214"
calc('floor(-2.7)') // "-3"
calc('clamp(150, 0, 100)') // "100"
```

::callout{type="info"}
All functions are **exact** except `sqrt` and **negative-exponent** `pow`, which are irrational/repeating and round to the division precision (`_precision`, default 50). Transcendental operations (`sin` / `log` / `exp`, etc.) are not supported by the BigInt rational-number model.
::

::calc-playground
```ts
calc('0.1 + 0.2 + 0.3') // "0.6"
calc('max(3, 5) * 2') // "10"
calc('clamp(150, 0, 100)') // "100"
```
::

### Formatting (`_fmt`)

Pass an [`IFormat` object](/guide/format-tokens) via `_fmt` to format the result:

```ts twoslash
import { calc } from '@wzo/calc'
// ---cut---
const price = 9.9
const qty = 3
calc(`${price} * ${qty}`, { _fmt: { decimals: 2 } }) // "29.70"
calc('1234567', { _fmt: { decimals: 2, thousands: true } }) // "1,234,567.00"
```

### Errors Throw (for computation)

Unknown functions, division by zero, invalid expressions — all **throw immediately**. `calc` is for computation, where errors should be visible:

```ts twoslash
import { calc, fmt } from '@wzo/calc'
// ---cut---
calc('1 + ') // ❌ throws: invalid expression
calc('1 / 0') // ❌ throws: division by zero

// Need a fallback (for display)? Use fmt:
fmt('1 + ') // "-" (fmt fallback on error)
fmt('1 / 0', { _error: 0 }) // 0
```

### Unit Mode

```ts twoslash
import { calc } from '@wzo/calc'
// ---cut---
calc('0.1% + 0.2%', { _unit: true }) // "0.3%"

// Without _unit, % means divide by 100
calc('0.1% + 0.2%') // "0.003"
```

### Debugging with `_debug`

`_debug: true` prints a step-by-step evaluation trace to the console, making it easy to inspect an expression:

```ts twoslash
import { calc } from '@wzo/calc'
// ---cut---
calc('1 + 3 * 2', { _debug: true })
// [calc] 1 + 3 * 2
//   · 3 * 2 = 6
//   · 1 + 6 = 7
//   = 7
```

To receive structured data instead of printing to the console, pass a function to `_debug` — it receives `{ expr, steps, value, result }`:

```ts twoslash
import { calc } from '@wzo/calc'
declare const log: (x: unknown) => void
// ---cut---
calc('1 + 2', { _debug: info => log(info) })
```
