# 注意事项

几个容易踩的边界，用之前扫一眼能省不少调试时间。

## 1. 计算出错会抛异常，只有 `fmt` 兜底

`calc` / `add` 等独立函数 / 聚合 / 链式出错（非法表达式、非法入参等）一律**直接抛异常**，需要你自己 `try/catch`。**只有展示用的 [`fmt`](/api/fmt)** 出错走 `_error` 兜底（默认 `'-'`）。

```ts twoslash
import { calc, fmt } from '@wzo/calc'
// ---cut---
calc('1 / x') // ❌ 抛错：未定义
fmt('1 / x') // "-"  ✅ 兜底，不崩页
```

::callout{type="info"}
口号：**计算用 `calc`（要错误可见），展示用 `fmt`（要稳、不崩页）。**
::

## 2. `calc` 不支持变量，用模板插值

`calc` 是纯算术求值，**没有变量功能**——把值直接写进表达式即可：

```ts twoslash
import { calc } from '@wzo/calc'
// ---cut---
const price = 9.9
const qty = 3
calc(`${price} * ${qty}`) // ✅ "29.7"
// calc('price * qty', { price, qty }) ← 旧写法，已不支持
```

## 3. 返回 `number` 的版本超 2^53 会丢精度

`add` / `sub` / `mul` / `div` 返回 `number`，结果超出 `Number.MAX_SAFE_INTEGER`（2^53）就会丢精度。金额、大整数请用**字符串版** `addStr` / `subStr` / `mulStr` / `divStr`，或用 `calc` / `fmt`：

```ts twoslash
import { add, addStr } from '@wzo/calc'
// ---cut---
add('9007199254740993', '1') // 9007199254740994 ← number，可能已不精确
addStr('9007199254740993', '1') // "9007199254740994" ✅ 字符串，不丢
```

## 4. 入参精度：值在传进来之前可能就坏了

精度敏感的值（金额、大整数）**用字符串传**。`number` 字面量在进函数之前就已经被 JS 浮点算坏了，库救不回来：

```ts
addStr(0.1 + 0.2, 0) // "0.30000000000000004" ← 入参已坏
addStr('0.1', '0.2') // "0.3" ✅
```

模板插值同理：`` calc(`${0.1} + ${0.2}`) `` 没问题（插的是字面量 `0.1`），但 `` calc(`${0.1 + 0.2}`) `` 会先用浮点算坏。

## 5. 配置：全局 `setConfig` 是单例 + 有时序

`setConfig` 改的是**全局单例**，必须在任何 `calc` / `fmt` 调用**之前**执行（通常放在 `main.ts`）。只想"某次除法 / 平均用别的精度、不污染全局"时，直接在末尾传 [`{ _precision }`](/api/config) 即可，没有时序顾虑。

## 6. 除法默认保留 50 位小数

无限循环小数（如 `1 / 3`）默认保留 **50 位**。要调整用 `_precision`（运算精度）或 `fmt` 的 `decimals`（显示位数）：

```ts twoslash
import { divStr, fmt } from '@wzo/calc'
// ---cut---
divStr('1', '3') // 50 位的 "0.333…"
divStr('1', '3', { _precision: 10 }) // "0.3333333333"
fmt('1 / 3', { decimals: 4 }) // "0.3333"
```
