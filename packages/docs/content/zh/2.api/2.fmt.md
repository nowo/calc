# `fmt(value, options?)`

**`fmt` 是 `calc` 的「展示版」**：用法一致、第一个参数同样**支持算术运算**，区别是 **出错走 `_error` 兜底而不是抛异常**——适合直接用于页面渲染，一条脏数据也不会让整页崩。

::callout{type="info"}
**展示用 `fmt`，计算用 [`calc`](/api/calc)。** `fmt` 出错兜底（返回 `_error`，默认 `'-'`）；`calc` 及其余函数（`add`/`calcSum`/`chain…`）出错直接抛，由你自己 try/catch。**只有 `fmt` 有兜底。**
::

## 签名

```ts
fmt(value: string | number | bigint, options?: IFmtOptions): string | number

interface IFmtOptions extends IFormat {
    _error?: string | number // 出错兜底值（默认取全局 _error，通常 '-'）
    _precision?: number // 参与运算时的除法精度（默认全局 _precision）
    _unit?: boolean // 启用 % 等单位运算（同 calc 的 _unit）
}
```

- 传 `string` ⇒ 当**算术表达式**求值后再格式化（`fmt('999.99 * 3')` ⇒ `'2999.97'`）
- 传 `number` / `bigint` ⇒ 直接格式化
- 出错 ⇒ 返回 `_error` 兜底（局部 `options._error` 优先于全局，默认 `'-'`）

格式化字段见 [`IFormat`](/guide/format-tokens)（如 `{ decimals: 2, thousands: true }`）。返回 `string`，`{ output: 'number' }` 时返回 `number`。

## 模板里直接用

```vue
<!-- 一条脏数据也不会让页面崩，自动显示兜底 '-' -->
{{ fmt(`${price} * ${qty}`, { decimals: 2, thousands: true }) }}
```

## 用法

```ts twoslash
import { fmt } from '@wzo/calc'
// ---cut---
fmt(1234567) // "1234567"
fmt(1234567, { thousands: true }) // "1,234,567"
fmt(1234567, { decimals: 2, thousands: true }) // "1,234,567.00"
fmt(0.5, { output: 'percent' }) // "50%"
fmt(1234567, { compact: true }) // "1.23M"
fmt(1234567, { compact: 'zh' }) // "123.45万"
fmt(0.5, { output: 'fraction' }) // "1/2"
fmt(1000, { output: 'scientific' }) // "1e+3"
fmt(5, { pad: 3 }) // "005"
```

## 输出为 number

`{ output: 'number' }`：

```ts twoslash
import { fmt } from '@wzo/calc'
// ---cut---
fmt(0.1234, { decimals: 2, output: 'number' }) // 0.12  ← number 类型
```

完整字段见 [数字格式化](/guide/format-tokens)。
