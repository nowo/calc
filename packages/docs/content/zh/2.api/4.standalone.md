# 独立函数

最直接的形式 —— 不构造表达式、不开链式，直接两数相加。

## 在线试一试

::calc-playground
```ts
add(0.1, 0.2) // 0.3   (number)
addStr('0.1', '0.2') // "0.3" (string，精确)
divStr('1', '3') // "0.333..."（最多 50 位）
```
::

## 返回 number 的版本

`add / sub / mul / div`：

```ts twoslash
import { add, div, mul, sub } from '@wzo/calc'
// ---cut---
add(0.1, 0.2) // 0.3
sub(1, 0.3) // 0.7
mul(3, 4, 5) // 60
div(10, 4) // 2.5

// 多参数会被 reduce
add(1, 2, 3, 4) // 10
mul(2, 3, 4) // 24
```

::callout{type="info"}
这几个函数内部仍走精度算术，但最终 `Number()` 一下。**结果在 Number 安全范围内时严格正确**，超出范围会丢精度。
::

### 用数组传参

`add / sub / mul / div`（以及 `addStr…` / `chainAdd…`）都是可变参数，可以用扩展运算符 `...` 把数组展开传入：

```ts twoslash
import { add, addStr, chainAdd } from '@wzo/calc'
// ---cut---
const nums = [0.1, 0.2, 0.3]
add(...nums) // 0.6
addStr(...nums) // "0.6"
chainAdd(...nums)() // "0.6"

// 一组金额求和（保精度）
const prices = ['9.99', '19.99', '4.95']
addStr(...prices) // "34.93"
```

::calc-playground
```ts
const nums = [0.1, 0.2, 0.3]
addStr(...nums) // "0.6"

const prices = ['9.99', '19.99', '4.95']
addStr(...prices) // "34.93"
```
::

## 返回 string 的版本（Str 后缀）

`addStr / subStr / mulStr / divStr` —— `Str` 后缀表示「返回高精度 `string`，不丢精度」：

```ts twoslash
import { addStr, divStr, mulStr } from '@wzo/calc'
// ---cut---
addStr('0.1', '0.2') // "0.3"
mulStr('0.1', '0.2') // "0.02"
divStr('1', '3') // "0.3333333333333333333333333333333333333333333333333"

// 大整数也精确
addStr('99999999999999999', '1') // "100000000000000000"
mulStr('9999999999', '9999999999') // "99999999980000000001"
```

## 什么时候用哪个

| 场景 | 用 |
| :--- | :--- |
| 算完直接进入业务逻辑（比较 / 存数据库等）| `add / sub / mul / div`（number）|
| 算完要显示给用户 | `addStr / subStr / mulStr / divStr`（string，配合 `fmt`）|
| 算完是个超大数 / 超精确小数 | 必须用 `Str` 后缀版，否则 Number 会丢 |

## 入参精度陷阱

上面讲的是**输出**：number 版结果超出 Number 安全范围会丢精度。但还有个更隐蔽的**输入**问题——

混用 number 和 string 完全合法（`add(0.1, '0.2')` 没问题），但要记住：**值在「传进函数之前」就可能被 JS 浮点算坏了，库再精确也救不回来。**

```ts
// number 字面量本身已经是坏的
addStr(0.1 + 0.2, 0) // "0.30000000000000004" ← 入参就是这个值
addStr(9007199254740993, 0) // "9007199254740992"  ← 超出安全范围，字面量已被舍入

// 用字符串传入，全程不丢精度
addStr('0.1', '0.2') // "0.3"
addStr('9007199254740993', '0') // "9007199254740993"
```

::warning
是否丢精度，**不取决于你混不混用 number / string，只取决于「精度敏感的那个值你写成了 number 还是 string」**。金额、大整数、或来自其他计算的中间结果，一律用字符串传入。
::
