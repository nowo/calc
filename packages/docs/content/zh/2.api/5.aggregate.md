# 聚合

直接对数组或对象数组做求和 / 求平均 / 最大 / 最小。结果都是 string。

## API

```ts
calcSum(keyOrArr, list?): string
calcAvg(keyOrArr, list?): string
calcMax(keyOrArr, list?): string
calcMin(keyOrArr, list?): string
```

## 在线试一试

::calc-playground
```ts
calcSum([1, 2, 3, 4]) // "10"
calcAvg([1, 2, 3]) // "2"
calcSum('price', [{ price: 10 }, { price: 20 }]) // "30"
calcMax([3, 10, 2]) // "10"
```
::

## 两种调用形式

### 1. 直接传数组

```ts twoslash
import { calcAvg, calcMax, calcMin, calcSum } from '@wzo/calc'
// ---cut---
calcSum([1, 2, 3]) // "6"
calcAvg([1, 2, 3]) // "2"
calcMax([3, 1, 2]) // "3"
calcMin([3, 1, 2]) // "1"
```

### 2. 传 key + 对象数组

```ts twoslash
import { calcAvg, calcMax, calcSum } from '@wzo/calc'
// ---cut---
const orders = [
    { price: 9.99, qty: 3 },
    { price: 19.99, qty: 2 },
    { price: 29.99, qty: 1 },
]

calcSum('price', orders) // "59.97"
calcAvg('price', orders) // "19.99"
calcMax('qty', orders) // "3"
```

## 配合 fmt 输出

```ts twoslash
import { calcSum, fmt } from '@wzo/calc'
declare const records: Array<{ amount: number }>
// ---cut---
const totalRaw = calcSum('amount', records) // "12345.6789"
fmt(totalRaw, { decimals: 2, thousands: true }) // "12,345.67"
```

## 空值处理

后端返回的数据里常夹带 `null` / `undefined`。四个聚合函数都会**自动跳过空值**，不会崩溃：

```ts twoslash
import { calcAvg, calcMax, calcSum } from '@wzo/calc'
// ---cut---
const list = [{ p: 1 }, { p: null }, { p: 3 }, { p: undefined }]

calcSum('p', list) // "4"
calcAvg('p', list) // "2"（分母只算非空项：4 / 2）
calcMax('p', list) // "3"

// 全为空值 ⇒ 兜底返回 "0"，不抛错
calcSum('p', [{ p: null }]) // "0"
```

::callout{type="info"}
注意 `calcAvg` 的分母是**非空项个数**——空值既不进分子也不进分母。
::

## 也可以用 addStr + 扩展运算符

简单求和 / 求积不一定要用聚合函数——`addStr` / `subStr` / `mulStr` 都是可变参数，可以直接用扩展运算符把数组展开传入：

```ts twoslash
import { addStr } from '@wzo/calc'
declare const orders: Array<{ price: number }>
// ---cut---
const nums = ['9.99', '19.99', '4.95']
addStr(...nums) // "34.93"

// 从对象数组取字段
addStr(...orders.map(o => o.price)) // "59.97"
```

::callout{type="warning"}
区别在于 **`addStr` 不跳过空值、出错抛异常**，而 `calcSum` 帮你兜住了空值（且用 `addStr` 而非 `add`，避免大数丢精度）：

```ts
addStr(...list.map(v => v.p)) // ❌ 抛错（数组含 null）
addStr(...list.map(v => v.p).filter(v => v != null)) // ✅ "4"，但要手动过滤
calcSum('p', list) // ✅ "4"，自动跳过空值
```
::

## 比 reduce 强在哪

`calcSum([0.1, 0.2, 0.3])` 严格返回 `"0.6"`。原生：

```js
[0.1, 0.2, 0.3].reduce((a, b) => a + b, 0) // 0.6000000000000001
```
