# 汇率换算

汇率是小数，金额又对精度敏感 —— 正是原生浮点最容易出错的地方。把金额和汇率以**字符串**传入，`@wzo/calc` 会让每一分钱都精确。

## 基础换算

```ts twoslash
import { calc } from '@wzo/calc'
// ---cut---
// 美元 → 人民币，汇率 7.2345
const usd = 250.75
const rate = 7.2345
calc(`${usd} * ${rate}`, { _fmt: { decimals: 2, thousands: true, rounding: 'round' } })
// 250.75 * 7.2345 = 1814.050875 → "1,814.05"
```

## 为什么要用它

一次换算在原生 JS 里就可能漂移：

```ts twoslash
import { fmt } from '@wzo/calc'
// ---cut---
19.99 * 1.1 // 21.989000000000004  ← 原生浮点
fmt('19.99 * 1.1', { decimals: 2, rounding: 'round' }) // "21.99"  ✅ 精确
```

## 经基准货币算交叉汇率

没有直接的 EUR→JPY 汇率？用链式经美元中转，把两段汇率串起来：

```ts twoslash
import { chainMul } from '@wzo/calc'
// ---cut---
// EUR → JPY 经 USD：×1.08（EUR→USD）、×150（USD→JPY）
const eur = 500
chainMul(eur, 1.08, 150)({ decimals: 0, thousands: true })
// 500 * 1.08 * 150 = "81,000"
```

## 多币种持仓汇总

先把每笔持仓换算到统一货币，再用 `calcSum` 求和（高精度，大数不漂移）：

```ts twoslash
import { calcSum, fmt, mulStr } from '@wzo/calc'
// ---cut---
const holdings = [
    { ccy: 'EUR', amount: 1200, toUsd: 1.08 },
    { ccy: 'GBP', amount: 800, toUsd: 1.27 },
    { ccy: 'JPY', amount: 150000, toUsd: 0.0067 },
]

// 每笔折算成美元（返回 string，精确）
const inUsd = holdings.map(h => ({
    ...h,
    usd: mulStr(String(h.amount), String(h.toUsd)),
}))
// EUR 1296 · GBP 1016 · JPY 1005

fmt(calcSum('usd', inUsd), { decimals: 2, thousands: true }) // "3,317.00"
```

## 反向汇率

反向汇率是无限循环小数 —— 自己决定保留几位：

```ts twoslash
import { rawDiv } from '@wzo/calc'
// ---cut---
// USD→CNY 是 7.2345，则 CNY→USD = 1 / 7.2345
rawDiv('1', '7.2345', 6) // "0.138227"
```

::callout{type="warning"}
汇率和金额一律以**字符串**传入（`'7.2345'`，而不是经过运算后的 `7.2345`）。汇率写成 `number` 字面量没问题，但若它来自上一步的浮点计算，在到达本库之前就已经被算坏了 —— 详见 [注意事项](/guide/caveats)。
::
